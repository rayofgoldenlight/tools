1. The Scripting Language

The scripting language looks like Lua, but it is compiled down to a tiny custom bytecode. Here is what you need to know:

  • Top-level only: You may only write var declarations and function update(dt) ... end at the top level. No nested functions or classes.
  • Numbers only: All variables are f32 floats. There are no strings, tables, or arrays. The only place you use a string is key_down("right").
  • No local variables: Every var is global. Declare all your variables at the top of the script.
  • No custom functions: Only function update(dt) is supported. You cannot define helper functions yet.
  • Control flow: if / elseif / else / end and while / do / end work exactly like Lua.
  • Operators: + - * /, comparisons == != < > <= >=, and logic and or not.
  • Comments: Use -- this is a comment (same as Lua).
var speed = 200
var score = 0

function update(dt)
    if key_down("right") then
        obj[0].x = obj[0].x + speed * dt
    end

    if score == 10 then
        obj[1].visible = 0
    end
end

2. Shapes, Objects, and Animations

Your game world is built from Shapes (the visuals) and Objects (the instances placed in the scene).

  • Create shapes in the Object Manager (Shapes tab): Circle, Rectangle, Text, or draw a vector shape freehand.
  • Place objects in the Objects tab: pick a shape, set position, rotation, and scale.
  • Animations are sequences of shapes played at a given FPS. Assign an animation to an object to override its static shape.

In your script, you control objects by index:

  • obj[0].x / obj[0].y -- Position
  • obj[0].rotation / obj[0].scale -- Transform
  • obj[0].visible -- Show/hide (1 or 0)
  • obj[0].value -- A number injected into Text shapes that contain {}
  • obj[0].val1 through obj[0].val4 -- Four custom slots per object (great for HP, timers, flags)

Behaviors let you write per-shape logic using self:

behavior 2
    self.x = self.x + 50 * dt
    if self.x > 800 then
        self.x = 0
    end
end

Every object whose base shape is Shape 2 will run this behavior each frame.

3. Audio Integration

Upload any audio file (WAV, MP3, OGG) in the Audio tab. It is automatically compressed to a tiny low-bitrate MP3 to keep your game file small.

Use the Sound ID shown in the list:

  • play_sound(0) -- Play sound 0 once.
  • loop_sound(0) -- Loop sound 0 forever (perfect for background music).
  • stop_sound(0) -- Stop sound 0.
  • set_volume(0, 0.5) -- Set sound 0 to 50% volume (0.0 = silent, 1.0 = max).

Tip: Rename your audio files in the editor with the Rename button so you remember which ID is which!

4. Built-in Functions and Variables

  • dt -- Delta time in seconds. Multiply all speeds by this!
  • key_down("name") -- Returns 1 if held, 0 if not. Supported names: left right up down space enter escape shift ctrl tab w a s d e f q r and any single letter or digit.
  • mouse_x / mouse_y -- Current mouse position on the canvas.
  • mouse_down() -- Returns 1 if the mouse button is held.
  • abs(x) -- Absolute value.
  • sign(x) -- Returns -1, 0, or 1.
  • play_anim(id) -- Play animation on the current behavior object.

5. Editor Shortcuts

  • Ctrl + S -- Force compile and reload.
  • Shift + G + U + I (hold all four) -- Toggle the debug HUD overlay (FPS, key display, stats).
  • Mode: Edit -- Drag objects on the canvas. Click to select.
  • Mode: Play -- Keyboard input goes to the game. Dragging is disabled.

When your cursor is inside the script editor, arrow keys and spacebar work normally for typing. The game only captures keys when the editor is not focused.

The vision for creating the best, fastest, and most accessible web-based game engine in the world.

The Vision

People playing and making fun games. Aiming for infinite (max) fun using no (min) energy and no (min) storage space. Core values for the editor is to be fun, fast, resource efficient, versatile, popular, and easy

Potential To Work On

  • Ultra-Compact Binary Formats: Transitioning core serialization to variable-length LEB128 encoding and Zstd compression. Have in-built sound maker system to allow good tiny-sized music to be made within editor
  • Advanced Superinstructions: Improving the Virtual Machine to execute complex, repetitive operations (like physics, bounds checking, and state machines) in single bytecode instructions.
  • High-Level Scripting Abstractions: Expanding the scripting language with built-in functions for spatial partitioning, vector mathematics, and switch-state logic to remove repetitive boilerplate.
  • Seamless Hosting and Discovery: Need to make a way to allow people to find and play games from others. Also, need to have it in such a way that it makes sense why the engine is popular
  • Intelligent Asset Pipelines: Implementing automatic vector geometry simplification and dynamic audio downsampling natively in the browser to ensure the entire game-engine system is efficient.

Engine Compilation Commands

Below are the core terminal commands used to compile the engine, its tools, and the WebAssembly runtimes.

# Build and run the offline rust toolchain
cargo run -p tools

# Compile the WebAssembly script compiler
wasm-pack build compiler_wasm --target web --out-dir ../web/pkg_compiler

# Compile the WebAssembly game runtime
wasm-pack build runtime --target web --out-dir ../web/pkg

# Test out the site locally (using Rust's basic-http-server)
# (From Root)
cd web
basic-http-server .

# Validate code correctness
cargo check

# Run the tests made
cargo test -p compiler
cargo test -p format
